기계가 직접 막아요

코드를 자동으로 검사하는 도구가 위험한 코드를 그 자리에서 잡아요. 원하면 GitHub에 올릴 때 아예 통과 못 하게 막을 수도 있어요.

기능 하나하나

설치하면, 프로젝트에 이런 게 생겨요.

추상적인 "좋아져요"가 아니라, 실제로 만들어지고 봐주는 것들이에요.
핵심은 문서를 한 파일에 몰아넣지 않고 여러 곳에 나눠 두고, 코드가 바뀌면 자동으로 따라 갱신한다는 거예요.

📁 내 프로젝트/ — /hi-vibe:init 한 번이면 이 문서 지도가 깔려요
  • CLAUDE.md프로젝트 전체 지도 — 개요·요구사항·폴더 구조
  • api/MODULE.md폴더마다 하나 — 그 안의 상세 설계·데이터·주의점
  • ui/MODULE.md…폴더가 늘면 문서도 폴더 옆에 같이 붙어요
  • handover.md세션 인수인계 — 다음 대화가 맥락 안 잃게. 대화 정리(compact) 때 자동 기록돼요
  • CHANGELOG.md변경 이력 — 뭐가 언제 왜 바뀌었나
🔍
중복 재구현 방지pre-write · ⚡ 자동

새 함수·파일을 만들기 전에 "이미 있는지" 먼저 찾아봐요.
AI가 세션마다 있던 걸 또 만드는 걸 막죠. 없을 땐 "스캔한 N개 파일에서 못 찾았어요"라고 범위까지 밝혀요 — 추측이 아니라 근거로.

🩹
에러 땜빵 금지즉시 감지 · ⚡ 자동

에러를 fallback으로 덮는 대신 원인부터 보게 해요.
에러를 몰래 삼키는 코드(빈 catch, except: pass)를 쓰는 순간 감지해서 알려줘요. 정말 의도한 거면 주석 한 줄로 통과.

📊
근거 없는 단정 금지규율 · ⚡ 자동

"아마 이럴 거예요"를 사실처럼 말하지 않게 해요.
수치·API는 확인하고 답하거나, 못 하면 "추정입니다"라고 밝혀요. (context7 연결 시 최신 공식 문서까지 자동 조회)

👀
남의 눈 설계 리뷰post-write --deep · ⚡ 자동

코드를 쓴 기억이 전혀 없는 깨끗한 AI가 방금 짠 걸 봐요.
체크리스트가 못 잡는 판단 — 과잉설계·불필요한 기능·숨은 결합 — 을 "이건 좀 과한데?" 하고 짚어줘요.

🧠
기억·인수인계·검색handover · recall · ⚡ 자동

대화가 정리(compact)될 때마다 진행상황을 자동 기록하고, 새 대화 시작하면 최근 인수인계를 다시 넣어줘요.
"예전에 왜 이렇게 했지?" 하면 기록을 뒤져 날짜·이유까지 답하고요. (20개 넘으면 자동 보관, 여러 터미널 동시에 써도 안 꼬여요)

🎯
오탐 방지규율·스캐너 · ⚡ 자동

"안 쓰는 코드"라고 잘못 짚는 걸 막아요.
프레임워크가 이름 없이 부르는 함수, 테스트, export default 컴포넌트, 아직 만드는 중인 코드…
이런 걸 죽은 코드로 오해하지 않게 오탐 사례를 인덱스로 관리하고, 새 오탐이 나오면 규칙·테스트로 박제해요.

🔬
구조 점검audit · 🖐 직접

이미 코드가 많은 프로젝트를 스캔해서, 똑같은 중복만이 아니라 "90% 비슷하게 또 만든" 함수·안 쓰는 코드·너무 큰 파일까지 짚어줘요.
전부 스캐너 증거로만 말하고, 지우기 전 "아직 만드는 중"인지 구분해요.

🛡️
기계 강제 (린트·CI)guards --ci · 🖐 직접

린트(ruff·eslint)·타입 검사·순환 의존 검사를 설치하고, 원하면 GitHub Actions로 push마다 검문해요.
복잡도·순환 위반은 빌드 실패로 막아요. 기존 설정은 안 덮고 물어보고 설치.

🩺
자가진단doctor · 🖐 직접

훅은 Claude Code를 방해하지 않으려고 조용히 실패하도록 설계돼서, python3가 없으면 티 안 나게 꺼질 수 있어요.
doctor가 훅 4종과 스캐너를 실제로 돌려서 ✅/❌로 진짜 작동하는지 확인해줘요.

정직함이 곧 기능이에요

여기까진 기계, 여기부턴 AI.

”전부 자동!”이라고 뭉뚱그리지 않아요. 기계가 확실히 하는 것과 AI가 알아서 하는 것을 나눠서 보여줘요.
불이 켜져 있으면 기계가 보장, 테두리만 있으면 AI에게 맡긴 거예요.

위험한 코드(숨는 에러·비밀번호)를 그 자리에서 잡기코드 저장하는 순간⚙ 기계가 확실히
대화가 정리되기 전에 진행상황 저장대화가 길어질 때⚙ 기계가 확실히
새로 시작할 때 규칙 다시 챙겨주기새 대화 시작할 때⚙ 기계가 확실히
뭔가 바꿨는데 기록 안 하면 알려주기대화 끝날 때⚙ 기계가 확실히
“만들어줘” 하면 이미 있는지 먼저 찾기새로 만들기 전🤖 AI가 알아서
“다 했어” 하면 점검하고 문서도 맞추기다 만든 후🤖 AI가 알아서
“검토해줘” 하면 다른 눈으로 과한 설계 잡기설계 살펴볼 때🤖 AI가 알아서
💡
🤖 “AI가 알아서”는 100% 보장은 아니에요.
AI가 깜빡할 수도 있어요. 그럴 땐 명령어(/hi-vibe:pre-write 같은 거)를 직접 눌러서 확실히 챙기면 돼요 — AI가 놓친 안전벨트를 손으로 채우는 셈이에요.

AI-coding seatbelt · A Claude Code plugin

Floor it with AI —
hi-vibe is your seatbelt.

The vibe-coding seatbelt for Claude Code.

When you let AI write your code, the same mistakes keep coming back — it rebuilds things that already exist, papers over errors, lets docs go stale… hi-vibe watches your back. No commands to memorize — just talk normally.

What it catches

AI-written code repeats the same mistakes.

The more convenient it is, the more quietly these pile up. hi-vibe catches them one by one.

Rebuilds
Rebuilds what already exists. On a fresh chat it forgets what it made and writes the same thing again.
Papers over
Papers over errors. It hides the symptom instead of fixing the cause, so bugs stay hidden.
Decides alone
Won’t ask when unsure. It just makes ambiguous calls on its own.
Makes it up
States guesses as fact. It presents unverified numbers as if they were official specs.
Docs drift
Docs stop matching the code. The code changed; the explanation stayed the same.

How · three layers

Blocked by structure, not by asking nicely.

The seatbelt is woven in three layers — gentle up top, machine-enforced at the bottom.

Layer one

Remembers for you

It auto-records what the project is, how far you got, and what changed — so a fresh chat never loses the thread.

Layer two

Teaches the AI the rules

Rules like “search before you build”, “fix the cause, not the symptom”, “don’t state guesses as fact” are planted into the AI so it polices itself.

Layer three

The machine blocks it

A code checker catches risky code on the spot — and if you want, it can fail the build before it ever reaches GitHub.

Feature by feature

Install it, and your project gets this.

Not a vague "things get better" — actual things it creates and watches.
The core idea: docs aren't dumped in one file; they're split across the project and auto-updated as the code changes.

📁 my-project/ — one /hi-vibe:init lays down this doc map
  • CLAUDE.mdThe whole-project map — overview, requirements, folder layout
  • api/MODULE.mdOne per folder — its detailed design, data, and gotchas
  • ui/MODULE.md…more folders, more docs living right next to them
  • handover.mdSession handover — so the next chat keeps context. Auto-recorded on compaction
  • CHANGELOG.mdChange history — what changed, when, and why
🔍
No duplicate reimplementationpre-write · ⚡ auto

Before you make a new function or file, it searches whether one already exists — so the AI stops rebuilding what it made last session. When there's none, it says "not found across N scanned files" — evidence, not a guess.

🩹
No band-aid error handlinginstant detect · ⚡ auto

Instead of burying an error under a fallback, it makes you look at the cause. Code that silently swallows errors (empty catch, except: pass) is flagged the moment you write it. Genuinely intentional? One comment lets it pass.

📊
No unfounded claimsdiscipline · ⚡ auto

It won't state "probably X" as fact. Numbers and APIs get verified before answering — or labeled "estimate" when they can't be. (With context7 connected, it even pulls the latest official docs.)

👀
Fresh-eyes design reviewpost-write --deep · ⚡ auto

A clean-context AI with zero memory of writing the code reviews what you just wrote. It catches the judgment calls a checklist can't — over-engineering, needless features, hidden coupling — with a plain "this looks like too much."

🧠
Memory, handover & recallhandover · recall · ⚡ auto

Auto-records your progress every time the chat is compacted, and re-injects the latest handover when a new chat starts. Ask "why did we do it this way?" and it digs the records for the date and reason. (Auto-archives past 20 entries; safe across multiple terminals.)

🎯
No false positivesdiscipline · scanner · ⚡ auto

It won't wrongly call live code "dead." Framework-registered functions, tests, export default components, work-in-progress code… are kept off the dead list by a false-positive index — and every new false positive gets pinned down with a rule and a test.

🔬
Structure checkupaudit · 🖐 manual

Scans a code-heavy project for not just exact duplicates but "90%-reimplemented" functions, unused code, and oversized files. Everything comes from scanner evidence — and it tells work-in-progress apart before suggesting a delete.

🛡️
Machine enforcement (lint · CI)guards --ci · 🖐 manual

Installs lint (ruff·eslint), type checks, and circular-dependency checks, and can gate every push via GitHub Actions. Complexity and cycle violations fail the build. It never overwrites your config — it asks first.

🩺
Self-diagnosisdoctor · 🖐 manual

Hooks are built to fail silently so they never block Claude Code — which means a missing python3 could turn them off unnoticed. doctor actually runs all 4 hooks and the scanner and shows ✅/❌ so you know they really work.

Honesty is the feature

Machine up to here, AI beyond.

We don’t hand-wave “it’s all automatic.” We separate what the machine guarantees from what the AI does on its own.
A lit lamp means guaranteed; a hollow one means it’s up to the AI.

Install · 30 seconds

Three lines. All you need is python3.

Paste these three lines in order inside Claude Code.
No heavy setup, no sign-up.

# 1) register hi-vibe
/plugin marketplace add jx-hxxx/hi-vibe
# 2) install the plugin
/plugin install hi-vibe@hi-vibe-marketplace
# 3) turn it on now (don’t skip!)
/reload-plugins
/hi-vibe:welcome

Start here if you’re not sure what to do.

/hi-vibe:doctor

Once after install — confirms it’s really on.

/hi-vibe:init

Once per project — turns hi-vibe on in that folder.

/hi-vibe:guards --ci

When you want — install the code checker so the machine blocks bad code.
